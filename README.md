# Chirpy API 🐦

Backend-сервис на Go для социальной сети (аналог Twitter), созданный в рамках обучения на Boot.dev. 

## 🚀 Основные возможности

- **REST API**: Эндпоинты для регистрации, логина и управления сообщениями (чирпами).
- **База данных**: PostgreSQL + `sqlc` для типобезопасной работы с SQL.
- **Аутентификация**: 
  - JWT Access/Refresh токены.
  - Хеширование паролей (bcrypt).
- **Безопасность**: Защита вебхуков Polka через API Key.
- **Функционал**:
  - Фильтрация "плохих" слов.
  - Сортировка и фильтрация чирпов по автору (Query params).

## 🛠 Стек технологий

- **Язык**: Go
- **БД**: PostgreSQL
- **Инструменты**: `sqlc`, `go-chi`, `golang-jwt`, `godotenv`.

## 📂 Структура проекта

Проект организован в соответствии со стандартами Go и требованиями Boot.dev:

- **`internal/database/`** — сгенерированный код `sqlc` для работы с PostgreSQL.
- **`internal/auth/`** — логика аутентификации: работа с JWT, хеширование паролей и токены.
- **`sql/schema/`** — SQL-миграции для создания таблиц пользователей, чирпов и токенов.
- **`sql/queries/`** — SQL-запросы, на основе которых `sqlc` генерирует Go-код.
- **`handlers.go`** — HTTP-хендлеры для всех эндпоинтов API.
- **`models.go`** — описание структур данных для JSON ответов.
- **`requests.http`** — готовые наборы запросов для тестирования API через REST Client.
